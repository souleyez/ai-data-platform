# AIDATE V0.1.1 发版说明（2026-03-18）

## 版本定位

这是在 V0.1 Demo 基础上的一次小版本迭代，重点不是新增大模块，而是把**资料接入后的用户反馈体验**补完整，并清理客户可见界面中的底层平台暴露信息。

适合用于：

- 内部演示
- 小范围客户试用
- 继续收集论文 / 技术文档场景反馈

---

## 本次重点更新

### 1. 上传 / 采集反馈增强（方案 A / MVP）

资料上传或采集完成后，对话框现在会提供更明确的结构化反馈：

- 成功提示
- 预解析标题
- 预解析摘要
- 推荐分类
- 推荐理由
- 单条 / 批量结果汇总

这能明显减少“传上去了，但系统到底怎么理解”的不确定感。

### 2. 上传反馈展示优化

进一步优化了资料接入后的预览细节：

- 优先显示原始上传文件名，而不是带时间戳的落盘名
- 标题提取更干净，去掉 markdown 标题符号等噪音
- 摘要会做轻量清洗，减少 markdown 痕迹

### 3. 客户可见界面去底层平台暴露

已清理客户可见 UI 中不适合对外展示的底层平台字样：

- 侧边栏品牌副标题
- 页面描述信息
- 对话区状态文案
- 分析链路状态展示

当前界面会尽量保持中性产品口径，不暴露底层实现来源。

---

## 本次未覆盖内容

以下仍不在 V0.1.1 范围内：

- 修改分类交互
- 一键确认入库
- 失败重试
- OCR / 复杂 PDF 增强解析
- 更复杂的自动归档策略

---

## 验收结论

本次版本已完成：

- API 测试通过
- API / Web 构建通过
- 实际完成单文件上传、批量上传、网页采集失败反馈等场景验收

当前可以作为一个**可演示、可试用的小版本**继续推进。
