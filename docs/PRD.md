# PRD - AI 数据分析中台

## 1. 产品定位

一个基于 OpenClaw 的本地化 AI 数据分析中台，面向企业内部使用。

核心价值：

- 统一接入文档、数据库、网页后台、API 等企业数据源
- 提供只读型智能问答与经营分析
- 生成可视化报表与结构化摘要
- 通过网页工作台让用户像使用 IM 一样提问

## 2. 目标用户

- 企业管理者
- 数据分析人员
- 运营人员
- 销售/客服/采购/交付团队
- 知识管理负责人

## 3. 核心原则

1. **只读优先**：不允许修改客户原系统
2. **本地部署优先**：系统安装在本地电脑或企业内网环境
3. **来源可追溯**：回答尽量附来源
4. **权限可控**：不同用户能看到的数据范围不同
5. **多源融合**：文档、数据库、网站后台统一分析

## 4. 核心场景

### 4.1 企业问答

- 这个月订单趋势如何？
- 哪些客户成交额下降了？
- 最近合同里有哪些付款风险？
- 技术文档主要涉及哪些模块？
- 商城订单退款率变化如何？

### 4.2 文档分析

- 扫描指定文件夹内 PDF / Word / Excel
- 技术文档自动摘要、归类、知识点提炼
- 合同自动抽取合同编号、金额、期限、付款节点、风险条款
- 论文/方案文档自动总结方法、结论、关键词

### 4.3 数据库分析

- 使用只读账户访问数据库
- 自然语言转只读 SQL
- 输出表格与图表
- 支持经营数据分析、订单分析、客户分析

### 4.4 网站后台只读采集

- 使用账号密码登录 ERP/OA/商城后台
- 导航到订单/报表/流程等页面
- 读取表格与详情数据
- 生成分析结果与可视化界面

## 5. MVP 范围

### 第一阶段（建议优先实现）

- 网页工作台
- IM 式问答
- 指定文件夹文档扫描
- PDF / 合同 / 技术文档解析
- 文档分类与结构化摘要
- 数据库只读查询
- 图表与表格输出
- 问答引用来源展示

### 第二阶段

- 网页后台只读抓取（ERP/OA/商城）
- 任务调度
- 自动日报/周报/月报
- 权限与审计

### 第三阶段

- 多租户/多组织
- 自定义分析模板
- 高级报表中心
- API 对外开放

## 6. 非目标

首版不做：

- 对原系统写入数据
- 自动提交表单
- 自动删除/编辑/保存后台数据
- 开放互联网公网多租户 SaaS

## 7. 功能模块

1. 对话中心
2. 文档中心
3. 数据源中心
4. 数据库分析中心
5. 网页采集中心
6. 报表中心
7. 权限中心
8. 审计日志

## 8. 成功标准

- 用户能通过聊天问到企业经营问题并获得可靠回答
- 回答中可展示引用来源与图表
- 合同与技术文档能自动结构化整理
- 数据库查询限定在只读与白名单范围
- 系统部署在本地后可稳定运行
