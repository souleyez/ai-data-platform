# 上传 / 采集后对话反馈增强 PRD（方案 A / MVP）

## 背景

当前用户在完成文件上传、链接采集或数据导入后，如果系统只返回简单成功状态或静默入库，用户难以判断系统是否正确接收并理解内容，也难以快速决定该数据应归入哪一类。

本需求希望在上传 / 采集完成后，在对话框中提供结构化反馈，增强可感知性与信任感。

## 目标

- 给出明确成功提示
- 展示预解析结果，帮助用户快速确认系统理解是否正确
- 给出推荐分类及推荐理由
- 支持单条与批量场景

## 本期范围（方案 A）

- 上传 / 采集成功提示
- 预解析内容展示（标题 + 短摘要）
- 推荐分类
- 推荐理由
- 单条 / 批量结果展示

## 不在本期范围

- 修改分类交互
- 一键确认入库
- 失败重试
- 精细化置信度展示
- 复杂按钮化卡片交互

## 主要交互

### 单条上传/采集成功

- 已成功接收 1 份文件，并完成预解析
- 文件名 / 来源
- 识别标题
- 内容摘要
- 推荐分类
- 推荐理由

### 批量上传/采集成功

- 已完成 N 项数据处理：成功 X 项，失败 Y 项
- 逐项展示成功对象的标题、摘要、推荐分类、推荐理由
- 对失败项展示失败原因，不展示伪造推荐结果

## 验收标准

1. 上传单个文件后，对话框能显示成功提示
2. 成功处理的数据对象能显示标题和摘要
3. 每个对象能显示推荐分类
4. 推荐分类有简短理由说明
5. 批量上传时能区分成功和失败项
6. 失败项不显示伪造推荐分类
